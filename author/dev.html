Switching to pure Android

Published: Sat 30 December 2023
By dev

In misc.

tags: Android

Over the past few months I have had many ideas for mobile applications. My development process usually followed this flow

1) Build the application in Expo

2) Expo does not provide a lot of flexibility or external library does not work

3) Build the application in React Native

4) React Native does not provide a lot of flexibility or external library does not work or build process takes way too long

5) Build the application as a Progressive Web Application (enable caching for all files, deploy via AWS Amplify, install on phone, end deployment)

I try with Expo and React Native because they are meant to be cross platform and easy to use. However it becomes very complex if I try to use third party libraries or want to use OS specific features. For example I tried to create Android application with widgets. There is a react native library that claims to support this but the documentation is not great, it requires me to modify android specific code, and a hello world equivalent runs into build failures. Atleast from Expo, I really liked the build process with Expo CLI and ExpoGo app.

PWAs tend to work but I can only build the equivalent of web apps. I only went down the rabbit hole of PWAs because the build process is very fast and if I enable the PWA to cache all files, I could stop the PWA host and it will still be able to run on a device.

Unfortunately the application ideas I have can not be productively completed with Expo, ReactNative, or PWAs. There are other cross platform tools like apache cordova or ionic, but I am very sure that these frameworks will run in to the same problem as Expo or ReactNative. They are great to build simple CRUD apps. I feel there are too many abstractions to build unique experiences and the cross platform capabilities quick go away when you want to add specific features or libraries.

So I decided to go switch to pure Android development. The Android ecosystem has greatly changed since I last coded on Android many years ago. The Jetpack Compose framework is very powerful and I like that we can quickly preview what the Compose components look like. The build process is also significantly cleaner and faster. I can easily pair my device via wifi (ReactNative requires cable. Expo has ExpoGo but still faced connectivity issues) and the application is built within minutes. If I had a better computer, the build could probably finish in under a minute.

The only downside of pure Android is that I lose the ability to create crossplatform builds. Although I still believe that Expo and ReactNative does not truly have crossplatform ability when you try build complex applications. And it is very unlikely that third party libraries can offer full support for Android and iOS.
